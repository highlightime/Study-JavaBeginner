## 06-1 객체지향 프로그래밍

### 객체

- 자신의 속성을 가지고 있음 && 식별 가능한 것
- ***속성=> 필드, 동작=>메소드*** 로 구성됨

### 객체 모델링

- 현실 세계의 객체를 소프트웨어 객체로 설계하는 것

### 메소드

- 객체들 사이 상호작용 수단
- 메소드 호출 : 객체가 다른 객체의 기능을 이용하는 것

### 객체간 관계

- **집합** 관계
- **사용** 관계
- **상속** 관계

### 클래스

- 객체를 위한 설계도
- 필드, 메소드 정의

### 인스턴스

- 클래스로부터 만들어진 객체

### 클래스 선언

- 소스파일은 클래스 선언을 담고있는 저장 단위, 클래스가 아님
- 소스파일을 컴파일 하면 바이트 코드 파일이 클래스 선언 개수만큼 생김

### 객체 생성

- new 연산자 : 힙 영역에 객체 생성 후 객체 번지수 리턴
- 주소를 참조 타입인 클래스 변수에 저장 =>  변수 통해 객체 사용 가능

### 클래스 두가지 용도 

- 실행용 : main 메소드를 제공하는 역할
- 라이브러리 클래스 : 다른 클래스에서 이용할 목적으로 설계



### 클래스 구성

- **필드** 
  - 객체의 고유 데이터, 부품 객체, 상태 정보 
  - 생성자와 메소드 전체에서 사용
  - 객체 소멸되지 않는 한 객체와 함께 존재
- **생성자**
  - 객체 생성시 초기화 담당
  - New 연산자로 호출되는 중괄호 블록
  - 필드 초기화 || 메소드 호출해 객체 사용할 준비
- **메소드**
  - 객체의 동작에 해당하는 중괄호 블록
  - 객체간 데이터 전달하는 수단

## 06-3 생성자

### 생성자

- new 연산자에 의해 호출되는 중괄호 블록
- 객체 생성시 호출됨
- 객체 초기화 담당

### 기본 생성자

- 선언 생략시 컴파일러가 자동으로 바이트 코드에 추가

### 명시적 생성자 선언

- 객체 다양한 값으로 초기화 목적

### 생성자 오버로딩

- 매개 변수를 달리하는 생성자 여러개 선언하는 것

### 다른 생성자 호출: this()

- 이유: 생성자 오버로딩이 많아질 경우 생성자간 중복 코드 발생
- 필드 초기화 내용 한 생성자(상세 생성자)에 집중적으로 작성, 나머지 생성자는 상세 생성자를 호출
- This() : 생성자의 철줄에서만 허용

```java
public class Car{
  String model, color;
  int max;
  
  Car(String model){
    this(model, "silver", 250);
  }
  Car(String model, String color){
    this(model, color, 250);
  }
  Car(String model, String color, int max){
    this.model= model;
    this.color= color;
    this.max= max;
  }
  
}
```



## 06-4 메소드

### 메소드

- 객체 동작에 해당하는 중괄호 블록
- 선언부, 실행 블록으로 구성

### 매개변수 개수 모를 경우

- 매개변수를 배열 타입으로 선언
- 문제점: 메소드 호출 전 배열 생성해야하는 불편함

```java
int sum2(int ...values){}
// 메소드 호출시 넘겨준 값의 수에 따라 자동으로 배열 생성, 매개값으로 사용
int result = sum2(1,2,3);
int result = sum2(1,2,3,4,5);
```



## 06-5 인스턴스 멤버와 정적 멤버

⚠️ 클래스는 객체의 설계도

### 인스턴스 멤버

**📌 정의**

- 객체마다 가지고 있는 멤버
- 객체 생성 후 사용할 수 있는 필드와 메소드

**참조 변수**

- 객체 외부에서 인스턴스 멤버에 접근하기 위해 사용

**this**

- 객체 내부에서 인스턴스 멤버에 접근하기 위해 사용

**사용**

```java
Car myCar = new Car();
myCar.myfield=10;
myCar.myMethod(60);
```





### 정적 멤버

**📌 정의**

- 클래스에 고정되어 객체들이 공유하는 멤버
- 객체 생성 없이 사용할 수 있는 필드와 메소드

**static**

- 정적 멤버 선언에 사용

**사용**

- 클래스가 메모리로 로딩되면 멤버 사용

```java
ClassName.myField;
ClassName.myMethod(x,y);
```





### ⚠️ 필드 선언 

**인스턴스**

객체마다 가지고 있어야할( = 객체마다 다른 ) 데이터 (색깔)

**정적**

객체마다 가지고 있을 필요 x 공용데이터 (파이)

 

### ⚠️ 메소드 선언

**인스턴스**

인스턴스 필드 포함한 메소드

**정적**

인스턴스 필드 없는 메소드

❌ 정적 메소드 선언시 인스턴스 멤버, 객체 자신의 참조인 this 사용 불가

✔️ 객체 먼저 생성 후 참조 변수로 접근해 인스턴스 멤버 사용 가능



### 싱글톤

**📌 정의**

- 전체 프로그램에서 단 하나의 객체만 생성되도록 보장한 객체

**생성**

☑️ 생성자에 private 👈 클래스 외부에서 new 연산자로 호출 안되도록 

☑️ 외부에서 호출 가능한 getInstance() 

```java
static ClassName getInstance(){
  return singleton;
}
```



### Final 

**📌 정의**

- 초기값이 저장되면 프로그램 실행 도중에 수정할 수 없는 필드값

**생성**

☑️ 필드 선언 시에 초기값 (단순값)

☑️ 생성자 매개값으로 초기값 지정 (복잡한 초기화 코드 필요, 객체 생성시 외부 데이터로 초기화)

⚠️ 객체마다 저장될 수 있고 생성자 매개값을 통해 여러가지 값을 가질 수 있다.



### 상수(static final)

객체마다 존재하지 않고 클래스에만 존재

초기값 저장되면 변경 불가

```java
static final double EARTH_RADIUS = 6400;
```



## 06-6 패키지와 접근 제한자



### 패키지

**📌 정의**

- 파일 시스템의 폴더와 유사
- 클래스의 일부분
- 클래스를 유일하게 만들어주는 식별자



**패키지 선언**

- 해당 클래스가 어떤 패키지에 속할 것인지

```java
package bigpack.smallpack;
```



**패키지 이름**

❌ 숫자로 시작 안 돼

❌ java로 시작 안 돼 👈 자바 표준 API 에서만 사용

✔️모두 소문자로 작성이 관례

✔️ 도메인 이름 역순으로 (com.samsung.projectname)



**default 패키지**

- 패키지 선언이 없는 클래스

✔️ JAVA8 이전에서 가능

❌ JAVA11 이후 컴파일 에러



**import 문**

- 패키지 선언과 클래스 선언 사이 작성

❌ 상위 패키지 import 했다고 하위패키지까지 import 되는 것 아님



### 접근 제한자

**📌 접근**

클래스 및 인터페이스 이들이 가지고있는 멤버의 접근

⭐️ public 외부 클래스가 자유롭게 사용

⭐️ protected 같은 패키지, 자식 패키지가 사용

⭐️ private 외부에서 사용 불가

⭐️ defualt 같은 패키지에 소속된 클래스에서만 사용



👉 private < default < protected < public



### Getter Setter

객체 필드를 객체 외부에서 직접적 접근 막는다 👈 객체의 무결정 

✔️ 필드는 외부에서 접근할 수 없게 막기 👉 private

✔️ 메소드는 공개 👉 메소드 통해 필드에 접근

**Setter**

매개값 검증해 유효한 값만 객체 필드로 저장 가능

```java
void setSpeed(double speed){
  if (speed<0){
    this.speed=0;
    return;
  }else{
    this.speed=speed;
  }
}
```



**Getter**

필드값 가공 후 외부로 전달 

```java
double getSpeed(){
  double km =speed*1.6;
  return km;
}
```



**읽기전용**

getter 메소드만 선언

setter 메소드가 private
